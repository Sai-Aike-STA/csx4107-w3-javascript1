JavaScript is commonly used to enhance the user experience while browsing the web.

It allows web pages to become interactive and dynamic, rather than purely static.

Technically, JavaScript works by manipulating the HTML and CSS on-the-fly — it can read information from the page, perform operations or calculations, and then update the page content or appearance instantly without needing to reload.

JavaScript is often served by the frontend server — sometimes as a static file, and sometimes dynamically generated by modern frameworks.

However, the execution always happens in the browser, not on the frontend server.

The server’s role is to provide the script, while the browser’s role is to process and run it.

JavaScript’s syntax style is quite like Java and other object-oriented programming (OOP) languages, with only a few key differences.

If you’re already familiar with languages like C++ or Java, you’ll find JavaScript relatively easy to understand and pick up quickly.

While JavaScript looks like Java, they are not the same — JavaScript is primarily used for scripting and event-driven programming in web environments, while Java is a compiled, class-based language for broader applications.

## Can JavaScript “print” data?

JavaScript can display data, but not in the same way that HTML directly renders content on a page.

Instead, JavaScript often outputs information to the browser’s console, which is part of the Developer Tools.

## Where to write JavaScript

JavaScript can be included in two main ways — external or internal:

**External JavaScript** – written in a separate .js file and linked to the HTML page using the `<script src="..."></script>` tag.

This is the recommended method for most projects because it keeps your code organized and easier to maintain.

**Internal JavaScript** – written directly inside the HTML document within a `<script>` tag, usually placed in the `<head>` or at the bottom of the `<body>` section.

Unless you have a specific reason (for example, a small inline script or quick test), it’s best to use external JavaScript files — just like how you use external CSS.

## Try-Out Example:

```html
<!DOCTYPE html>
<html>
    <head>
        <title>Java Script Example 01</title>
    </head>
    <body>
        <script>
            console.log("Hello World...")
        </script>
    </body>
</html>
```

To observe the output:

Open your browser’s Developer Tools (usually by pressing F12 or Ctrl + Shift + I).

Go to the Console tab.

Refresh the page (F5) — you’ll see the message appear there!

The console is where developers test small pieces of code and check values while debugging JavaScript.

Because JavaScript is an interpreted script that runs together with the HTML, it is executed immediately as the page loads.

This means that any JavaScript placed in the `<head>` section runs before the page content is fully displayed, while scripts placed at the bottom of the `<body>` run after the page content has been loaded.

For most practical cases, it’s recommended to put your `<script>` tag just before the closing `</body>` tag — this ensures your page content loads first before the JavaScript executes.

## Statement and Expression

In JavaScript, statements are instructions that tell the computer to do something, while expressions produce a value.

Example:

```javascript
let x = 10;     // statement
x + 5;          // expression (produces 15)
```

## Arithmetic Operators

Arithmetic operators perform arithmetic on numbers (literals or variables).

| Operator | Description |
|---|---|
| + | Addition |
| - | Subtraction |
| * | Multiplication |
| ** | Exponentiation (ES2016) |
| / | Division |
| % | Modulus (Remainder) |
| ++ | Increment |
| -- | Decrement |

All of the arithmetic operators required two operands except the increment and decrement operators.

The increment/decrement operators can locate in front of the operand (prefix) or behind the operand (suffix)

Example:

```javascript
const a = 10;
const b = 3;

console.log("a =", a);
console.log("b =", b);

// Addition
console.log("Addition (a + b):", a + b);

// Subtraction
console.log("Subtraction (a - b):", a - b);

// Multiplication
console.log("Multiplication (a * b):", a * b);

// Division
console.log("Division (a / b):", a / b);

// Modulus (remainder)
console.log("Modulus (a % b):", a % b);

// Exponentiation
console.log("Exponent (a ** b):", a ** b);

// Increment / Decrement
let x = 5;

console.log("x =", x);
console.log("x++ =", x++);   // returns 5, but becomes 6
console.log("after x++ =", x);
console.log("--x =", --x);   // becomes 5
```

Logical operators are used to combine Boolean expressions.

## If statement

There are three possible if statement structures:

```javascript
if (//condition//) { … }

If (//condition//) { … } else { … }

If (//condition//) { … } else if (//condition//) { … } [else if { … }] [else { … }]
```

For the if-else-if structure, note that there can be any number of "else if" and may or may not end up with just else statement.

Example:

```javascript
if (x == 10) {
    console.log("1")
}

if (x == 10) {
    console.log("1")
} else {
    console.log("2")
}

if (x == 10) {
    console.log("1")
} else if (x == 20) {
    console.log("3")
} else if (x == 30) {
    console.log("4")
} else {
    console.log ("2")
}
```

## Switch-case statement

The switch syntax is as follow:

```javascript
switch (//value//) {
    case <constant 1> : /* codes */
        break;
    case <constant 2>: /* codes */
        break;
    …
    default: /* codes */
}
```

How it work:

The switch will take the value in its parenthesis to compare. The value can be literal or variable or expression

It compares the given value with the defined constant of each case.

If the value matched the constant (equal), it process the codes defined after colon (:)

It will process all the codes until it find a break keyword. This means that, it can process over to next constant case if it don't find the break keyword!!

If the value don't match to any constant, it use default case.

If no case match the value and there is no default it just skip the switch-case.

The codes follow a case can be grouped with backets { … } or can be written without grouping.

Example:

```javascript
let x = 10;

switch (x) {
    case 5:
        console.log("1");
        break;
    case 10:
        console.log("2");
        break;
    case 15:
        console.log("3");
        break;
    default: {
        console.log("4");
        console.log("No matched");
    }
}
```

## Ternary

The ternary is as a short cut of if - else statement for assigning a value to a variable.

Example:

```javascript
let x = (y == 10) ? 20 : 30;
```

Equivalent to

```javascript
let x;

If (y == 10) {
    x = 20;
} else {
    x = 30;
}
```

## Iteration Statement

There are 2 major iteration statements:

While statement

For statement

## Whie statement

The meaning can be simple interpreted like: do the following statements as long as the condition is true.

There are two type of while statements:

While .. Do - the statements may never be processed.

Do .. While - the statements will be processed at least once.

Example:

```javascript
let x = 10;
let y = 10;

// Do .. While
do {
    console.log("do..while: x = ", x);
    x++;
} while (x < 10);

// While .. Do
while (y < 10) {
    console.log("while..do: y = ", y);
    y++;
}
```

## For Statement

There are about three types of for statements. However, two of them will be discussed later when we discuss about Array in JavaScript.

The common for statement syntax is:

```javascript
for (<initial>; <while condition>; <update>) { … }
```

The simple interpret of for statement is: Begin with the initial value, as long as the condition is true, process the statements and at the end of the process, update the value.

Example:

```javascript
for (let i = 0; i < 10; i++) {
    console.log("i = ", i);
}
```

This for statement can be translated in while-do as:

```javascript
let i = 0;
while (i < 10) {
    console.log("i = ", i);
    i++;
}
```
